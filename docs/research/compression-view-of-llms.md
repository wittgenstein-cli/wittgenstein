# Compression View of LLMs

**Type:** Long-form theory note.

## Core Argument

Language models are implicit neural compressors. Wittgenstein is architected around this fact: the LLM compresses intent into a compact symbolic IR, and the codec expands that IR into a real file. This is not a metaphor — it is an architectural decision with measurable cost consequences.

## Shannon and the Entropy of Text

Shannon's source coding theorem establishes the fundamental limit: the minimum average description length of a message is its entropy. A compressor that achieves the entropy of a source is, by definition, modeling that source perfectly. Language models trained on next-token prediction are doing exactly this — they are approximating the conditional entropy of the next token given all prior tokens. The better the model, the closer it gets to the true entropy of human language.

The Hutter Prize makes this explicit: the winner of a 100 MB Wikipedia compression contest is, in a precise information-theoretic sense, the best language model. Fabrice Bellard's NNCP (2021) demonstrated that a small transformer trained on text outperforms classical compressors like gzip or bzip2 on natural language corpora. The connection runs both ways: Deletang et al., "Language Modeling Is Compression" (2023, DeepMind), showed formally that arithmetic coding with a language model as the probability estimator achieves near-optimal compression on text, and that the model's perplexity directly bounds the compression ratio. GPT-3 (Brown et al., 2020) noted this connection in passing; Deletang et al. elevated it to a central result.

## The Multimodal Implication

If the LLM is a text compressor, it is strongest when its outputs stay in the domain it was trained to compress: compact symbolic descriptions. Asking a language model to emit 786,432 raw pixel values for a 512×512 RGB image is asking a text compressor to output uncompressed binary data. The model has no statistical advantage there — pixels are not text, and the joint distribution of pixel values is not something a language model has learned to model efficiently.

Asking the same model to emit a 60-token JSON scene specification is the opposite: the model is being asked to compress its semantic understanding of an image intent into a compact, structured description. This is precisely the task the model's training has optimized it for.

## The Codec as Decompressor

The structured IR is the compressed representation. The codec is the decompressor. In Wittgenstein's image path, the LLM emits a scene JSON (palette, composition, style fields) of roughly 60 tokens. The adapter MLP translates that into VQ codebook indices. The frozen decoder reconstructs the image from those indices. The full pipeline is: compress intent into IR → expand IR into file.

This framing clarifies why the adapter is small. The adapter does not need to learn image semantics from scratch. The semantic compression has already happened inside the LLM. The adapter only needs to learn the interface between two representation spaces — text embeddings and codebook indices — which is a much lower-dimensional problem.

## Practical Cost Consequence

Token cost in API-hosted LLMs scales directly with token count. A 1024×1024 RGB image contains 1,024 × 1,024 × 3 = 3,145,728 scalar values. Even with 8-bit quantization and a fixed vocabulary, direct pixel emission would require millions of tokens per image. A 60-token scene spec costs roughly 1/52,000 of that budget. For a 512×512 image at VQ compression (16× spatial downsample), the discrete token sequence is 32×32 = 1,024 indices — still 17× longer than the scene spec, and those tokens are generated by the decoder, not the LLM.

The compression view predicts this cost asymmetry exactly: keep the LLM doing what it is good at (compressing into symbols), delegate expansion to a specialized decoder.

Thesis

Agent = Model + Harness—extended with modality codecs.

Industry harnesses handle tools, memory, and control flow. Wittgenstein adds a clear seam: natural structured IR plus deterministic renderers and optional tiny adapters. Post-training can move to the harness bundle instead of the base model when that trade-off wins.

The LLM stays the planner. The harness owns routing, schema preambles, validation, retries, budgets, sandboxed execution, and traces under artifacts/runs/.

Architecture

Five explicit layers—so work lands in the right package.

Same decomposition as the monorepo: runtime, IR, decoder, adapter, distribution.

L1
Harness / runtime — router, retry, seed, budget, telemetry, sandbox.
L2
IR / codec — zod schemas, prompt contracts, parse only structured success paths.
L3
Renderer / decoder — IR to bytes; frozen decoders in-bounds, generators out-of-scope.
L4
Optional adapter — small learned translators (e.g. scene → latents) shipped beside codecs.
L5
Packaging — CLI, install scripts, skills, docs as system of record.

Sole image path (locked)

Semantic JSON → adapter → frozen decoder → PNG.

No SVG/HTML pixel hacks in the product path: the model emits a validated scene spec; the harness expands, adapts to latent codes, decodes to raster, then packages a real file. Decoder ≠ generator; every run writes a trace.

User prompt → schema preamble + JSON contract → LLM (planner) → parse (zod) → expand → adapter (latents) → decoder (frozen) → packageRasterAsPng → artifact + manifest
